# 이미지시스템 Report

> 작성일: 2026-06-14
> 대상: `index.html` 이미지시스템 기능
> 상태: 구현 완료 · 사용자 테스트 대기

## Executive Summary

| 관점 | 결과 |
|---|---|
| Problem | Claim 화면에 사진·서류 증빙 관리와 고객 발송 기능이 없었다. |
| Solution | 좌측 사이드바로 화면을 분리하고 사고건별 7폴더 이미지 시스템 + 문자 발송을 추가. |
| Value Delivered | 사고건당 사진·서류를 폴더로 관리(4건 시드 6/7/9/8장), 보안링크/직접첨부 문자 발송 + 발송 이력 기록. |
| Core Value | 증빙 관리와 고객 커뮤니케이션을 ERP 내부로 통합, 분쟁 대비 이력 확보. |

## 1. 요약

`조치대상 관리`와 분리된 `이미지시스템` 화면을 좌측 사이드바로 추가했다. 사고건별 7개 폴더(사고/고객/수리전/수리완료/청구서류/종결/기타)로 이미지를 관리하고, 고객 요청 시 보안 링크 또는 직접 첨부(MMS)로 문자 발송하며 발송 이력을 남긴다.

## 2. 완료된 항목

- 좌측 사이드바 네비(조치대상 관리/이미지시스템) + 화면 전환(`switchView`)
- 확장 메뉴(대시보드/정산관리) Pro 비활성 스캐폴드
- 사고건 선택 리스트(검색, 총 장수 배지)
- 7개 폴더 탭(폴더별 장수, 빈 폴더 강조)
- SVG 플레이스홀더 썸네일(파일명·촬영일·출처 칩) + 다중 선택
- 고객 문자 발송 모달(보안링크/직접첨부 토글, 폰 미리보기, 알림톡/문자)
- 고객 발송 이력 섹션(누가·언제·방식·장수·파일명)
- 반응형(사이드바 축소, 이미지 레이아웃 1단)

## 3. 주요 구현 포인트

- 레이아웃을 `.app-shell`로 감싸 기존 화면을 `#viewClaims`로 보존 → 회귀 방지.
- 데이터: `IMAGE_FOLDERS`, `FOLDER_KIND`, `CLAIM_IMAGES`(4건 시드), `imageSendLog`.
- 발송 UI는 기존 phone-preview/모달/`formatNowStamp` 재사용 → 신규 CSS 최소화.
- 이벤트는 `#imgMain` 위임으로 재렌더 후에도 폴더/썸네일/액션 동작.

## 4. 검증 상태

- `node --check` 통과, 자료구조 검증(폴더 7개·키 정합·총 장수) 통과.
- `sendImages` 로그 적재(LINK/MMS 2건) 정상.
- HTML 구조 태그 균형 확인.
- 브라우저 실제 클릭은 사용자 테스트 예정.

## 5. 반드시 유지할 테스트 시나리오

1. 사이드바 `이미지시스템` 전환 → topbar "이미지 시스템".
2. `CLM-2026-0004` → `수리전사진`(3장) 그리드.
3. 이미지 2장 선택 → `문자 발송` → 보안/MMS 토글 → 발송 → 토스트 + 이력 1건.
4. `조치대상 관리` 복귀 시 검색/Flow/빠른액션 모달 정상.

## 6. 후속 권장 작업

- 라이트박스 확대 뷰어 실제 구현(현재 데모 토스트)

- 채널COSS 자동 수집 사진 연계, 수신 확인, 종결 패키지 일괄 발송
