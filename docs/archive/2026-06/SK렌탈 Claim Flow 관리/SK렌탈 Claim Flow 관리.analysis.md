## Missing Items

- [ ] 토스트 영역에 `aria-live="polite"`가 없다.
  - 설계 문서 15번 접근성 기준에서 요구한 항목이다.
  - 현재 `#toastWrap`과 동적으로 생성되는 `.toast` 모두 `aria-live` 속성을 갖지 않는다.

- [ ] 설계 명세의 `renderMetrics()` 전용 함수가 없다.
  - 보조 성과 지표 자체는 HTML에 표시되어 있으므로 사용자 경험상 큰 문제는 없다.
  - 다만 설계의 함수 단위와 정확히 일치하지는 않는다.

## Changed Items

- [ ] Flow 카드 클릭 시 조치유형 필터를 유지하지 않고 초기화한다.
  - 설계 문서 10번은 Flow 카드 클릭 시 `selectedActionType`을 유지한다고 정의한다.
  - 현재 구현은 `activeType = null`로 초기화한다.
  - 실제 업무 UX에서는 단계 전환 시 필터 초기화가 더 자연스러울 수 있지만, 설계 일치 관점에서는 차이다.

- [ ] 설계의 HTML 클래스명과 실제 클래스명이 다르다.
  - 설계 예시: `.app-shell`, `.page-header`, `.flow-section`, `.metric-strip`, `.workbench`, `.action-tabs`, `.claim-list-panel`, `.quick-panel`
  - 실제 구현: `.topbar`, `.container`, `.flow-row`, `.kpi-line`, `.workspace`, `.tabs`, `.list-panel`, `.panel`
  - 구조와 역할은 일치하지만, 문서의 명명과는 다르다.

## Recommendations

1. Report 단계로 진행 가능하다. 현재 일치율 92%로 기준선 90%를 넘었다.
2. 접근성 품질을 높이려면 `#toastWrap`에 `aria-live="polite"`를 추가한다.
3. 설계와 행동을 엄격히 맞추려면 Flow 카드 클릭 시 `activeType` 초기화를 제거한다.
4. 장기 유지보수를 중시한다면 보조 성과 지표 렌더링을 `renderMetrics()` 함수로 분리한다.
5. 클래스명까지 설계 문서와 맞출 필요가 있는지 결정한다. 현재는 구조적 역할이 일치하므로 필수 수정은 아니다.
