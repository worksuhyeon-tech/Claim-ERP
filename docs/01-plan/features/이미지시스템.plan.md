# 이미지시스템 Plan

> 작성일: 2026-06-14
> 프로젝트 레벨: Starter
> 구현 방식: 단일 정적 HTML 데모 (`index.html`)
> 현재 상태: 구현 완료 (Plan/Design/Do/Check 반영)

## Executive Summary

| 관점 | 내용 |
|---|---|
| Problem | Claim 화면이 '조치대상 관리' 한 화면뿐이라, 사고보상 실무의 핵심인 사진·서류 자산 관리와 고객 발송을 할 수 없다. |
| Solution | 좌측 사이드바로 화면을 분리하고, 사고건별 7개 폴더형 이미지 시스템 + 고객 문자 발송(보안링크/직접첨부) 기능을 추가한다. |
| Function/UX 효과 | 담당자가 한 화면에서 사고건의 사진·서류를 폴더별로 확인하고, 고객 요청 시 즉시 문자로 발송하며, 발송 이력으로 분쟁에 대비한다. |
| Core Value | 사고보상 처리의 증빙 관리와 고객 커뮤니케이션을 ERP 안으로 통합한다. |

## Context Anchor

| 항목 | 내용 |
|---|---|
| WHY | 사고보상은 사진·서류 증빙과 고객 발송이 업무의 절반. 기존 화면엔 이 기능이 없었다. |
| WHO | SK렌탈 Claim 담당자, 운영 관리자 (데모 시연 대상: 임원·영업) |
| RISK | 단일 HTML 비대화 / 기존 '조치대상 관리' 화면 회귀 / 민감 이미지 노출 |
| SUCCESS | 사이드바 전환·폴더별 이미지·문자 발송·발송 이력이 동작하고 기존 화면이 깨지지 않음 |
| SCOPE | 단일 `index.html`, 바닐라 JS, 외부 네트워크 무의존, 실제 발송/저장소 없음 |

## 1. 목표

`조치대상 관리`와 분리된 `이미지시스템` 화면을 좌측 사이드바로 추가하여, 사고건별 사진·서류를 폴더로 관리하고 고객이 요청하면 바로 문자로 발송할 수 있게 한다.

## 2. 범위

### In Scope

- 좌측 사이드바 네비게이션(조치대상 관리 / 이미지시스템) + 화면 전환
- 사고건별 이미지 폴더 7종: 사고사진, 고객사진, 수리전사진, 수리완료사진, 청구관련서류, 종결, 기타
- 사고건 선택(검색 포함) → 폴더 탭(장수) → 이미지 그리드(다중 선택)
- 이미지 표현: 인라인 SVG 플레이스홀더(파일명·촬영일·출처)
- 고객 문자 발송: 보안 만료 링크 / 직접 첨부(MMS) 병행 (기존 phone-preview UI 재사용)
- 고객 발송 이력 기록(분쟁 대비)
- 향후 메뉴(대시보드/정산관리)는 'Pro' 비활성 스캐폴드
- 단일 `index.html` 유지, 외부 프레임워크/서버/DB 없음

### Out of Scope

- 실제 이미지 업로드/저장소, 실제 MMS/알림톡 발송
- 채널COSS/AOS/SK전산 실연동
- 라이트박스 확대 뷰어 실제 구현(데모 토스트로 대체)
- 권한/마스킹/워터마크, 수신 확인

## 3. 사용자/시나리오

- 담당자가 좌측 `이미지시스템` 클릭 → 사고건 선택 → 폴더별 사진·서류 확인
- 고객이 "내 사고 사진 보내달라" 요청 → 이미지 선택 → `문자 발송` → 보안링크 또는 직접첨부로 발송
- 분쟁 발생 시 `고객 발송 이력`에서 누가·언제·무엇을·어떤 방식으로 보냈는지 확인

## 4. 성공 기준 (Success Criteria)

- SC1: 좌측 사이드바로 '조치대상 관리' ↔ '이미지시스템' 화면이 전환되고 topbar 화면명이 바뀐다.
- SC2: 사고건 선택 시 7개 폴더 탭과 폴더별 장수가 표시된다.
- SC3: 폴더의 이미지가 SVG 썸네일(파일명·촬영일·출처)로 보이고 다중 선택된다.
- SC4: 선택 이미지를 `문자 발송`하면 보안링크/직접첨부 토글·미리보기가 동작하고, 발송 시 토스트 + 발송 이력 1건이 추가된다.
- SC5: '조치대상 관리'로 돌아가도 기존 기능(검색·Flow·빠른액션 모달)이 정상 동작한다.
- SC6: 단일 `index.html` 구조와 오프라인(외부 네트워크 무의존)을 유지한다.

## 5. 리스크

| 리스크 | 대응 |
|---|---|
| 단일 HTML 비대화 | 이미지 데이터(`CLAIM_IMAGES`)와 렌더 함수 분리, 폴더/플래그 기반 렌더 |
| 기존 화면 회귀 | 레이아웃을 `.app-shell`로 감싸되 기존 `#viewClaims` 마크업/이벤트 보존, 회귀 시나리오 유지 |
| 민감 이미지 노출 | 보안 만료 링크 발송 옵션 + 발송 이력 로깅 |
| 실제 이미지 부재 | 인라인 SVG 플레이스홀더로 오프라인 표현 |
