# 이미지시스템 Design

> 작성일: 2026-06-14
> 기준 문서: `docs/01-plan/features/이미지시스템.plan.md`
> 선택 아키텍처: Option C — Pragmatic Balance (기존 자산 최대 재사용 + 명확한 뷰 분리)

## Context Anchor

| 항목 | 내용 |
|---|---|
| WHY | 사고보상은 사진·서류 증빙과 고객 발송이 업무의 절반 |
| WHO | SK렌탈 Claim 담당자, 운영 관리자 |
| RISK | 단일 HTML 비대화 / 기존 화면 회귀 / 민감 이미지 노출 |
| SUCCESS | 사이드바 전환·폴더 이미지·문자 발송·발송 이력 동작, 기존 화면 유지 |
| SCOPE | 단일 `index.html`, 바닐라 JS, 오프라인 |

## 1. 화면 구조

```
.app-shell (flex)
├─ .side-nav (좌측 레일, sticky 100vh)
│   ├─ 브랜드(C / CLAIM ERP)
│   ├─ [메뉴] 조치대상 관리 · 이미지시스템
│   └─ [확장 Pro] 통합 대시보드 · 정산 관리 (disabled)
└─ .app-main (flex:1)
    ├─ topbar (현재 화면명 #viewName + user-chip)
    └─ main.container
        ├─ section#viewClaims (기존 조치대상 관리 화면 전체)
        └─ section#viewImages (신규 이미지시스템, hidden 기본)
            └─ .img-layout (grid 280px / 1fr)
                ├─ aside.img-claims (사고건 검색 + 리스트)
                └─ section.img-main (요약 + 폴더탭 + 그리드 + 액션바 + 발송이력)
```

- 화면 전환은 `.view[hidden]` 토글로 처리(SPA 라우팅 없음).
- 반응형: 980px↓ 사이드바 아이콘 레일 축소, `.img-layout` 1단 적층.

## 2. 좌측 사이드바

- 네비 항목은 `<button class="nav-item" data-view="claims|images">`.
- 활성 항목 `.active`(블루 soft), 비활성 확장 메뉴 `.disabled` + `Pro` 칩.
- `switchView(name)`이 `.view` hidden, `.nav-item.active`, topbar `#viewName` 갱신, images 진입 시 `renderImageView()` 호출.

## 5. 렌더링 설계

| 함수 | 역할 |
|---|---|
| `renderImageView()` | 첫 진입 시 첫 사고건 자동 선택 후 리스트+본문 렌더 |
| `renderImgClaimList()` | 이미지 등록된 사고건 리스트(검색 필터 `imgScopedClaims`, 총 장수 배지) |
| `renderImgMain()` | 선택 건 요약 + 폴더 탭(장수/빈폴더 강조) + 그리드 + 액션바 + 발송이력 |
| `renderImageThumb(im, kind)` | SVG 썸네일 카드(사진/문서) + 파일명·촬영일·출처 칩 + 선택 체크 |
| `selectImgClaim(id)` | 사고건 전환(폴더 초기화, 선택 해제) |

- 이미지/문서 아이콘은 인라인 SVG 상수(`ICON_PHOTO`/`ICON_DOC`/`ICON_CHECK`).
- 사진 영역은 체크무늬 배경 + 아이콘으로 "이미지 자리"임을 표현.

## 6. 고객 문자 발송 설계

- `openImageSendModal(claim, images)` → 기존 모달 셸(`#actionModalRoot`) + phone-preview UI 재사용.
- 본문 좌측: 발송 방식 토글(`보안 링크`/`직접 첨부(MMS)`) + 첨부 자료 목록.
- 본문 우측: 폰 미리보기 — `link` 모드는 만료 안내 + 보안 링크 박스, `MMS` 모드는 이미지 첨부 칩. 알림톡/문자 채널, 메시지 편집.
- `sendImages(claim, images, mode, channel)`: `imageSendLog`에 적재(`formatNowStamp` 사용) → 선택 해제 → 모달 닫기 → 토스트 → `renderImgMain()`.

## 7. 이벤트 설계

- `.nav-item[data-view]` 클릭 → `switchView`.
- `#imgSearchInput`/`#imgSearchClear` → `applyImgSearch`(검색, 선택 건 유효성 보정).
- `#imgClaimList` 위임 클릭 → `selectImgClaim`.
- `#imgMain` 위임 클릭 → 폴더 탭/썸네일 선택/액션바(`문자 발송`·`다운로드`·`확대보기`). 재렌더 후에도 위임으로 동작.

## 8. 테스트 시나리오

1. 사이드바 `이미지시스템` → 화면 전환, topbar "이미지 시스템".
2. `CLM-2026-0004` 선택 → 폴더 탭(장수) → `수리전사진`(3장) 그리드 확인.
3. 이미지 2장 선택 → 액션바 `문자 발송` → 보안링크/직접첨부 토글·미리보기 → 발송 → 토스트 + 발송 이력 1건.
4. `조치대상 관리` 복귀 → 검색/Flow/빠른액션 모달 정상.

## 9. 구현 제약

- 단일 `index.html`, 바닐라 JS, 외부 네트워크/패키지 무의존.
- 기존 `#viewClaims` 마크업·이벤트·CSS 보존(회귀 방지).
- 실제 발송/저장 없음(데모 토스트·로컬 상태).

## 10. 후속 확장(설계만)

권한별 마스킹·워터마크, 라이트박스 확대 뷰어, 업로드 무결성/변경 이력, 채널COSS 자동 수집 연계, 수신 확인, 종결 패키지 일괄 발송, 발송 요청 근거 자동 첨부.
