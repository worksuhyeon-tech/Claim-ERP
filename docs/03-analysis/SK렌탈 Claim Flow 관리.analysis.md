# SK렌탈 Claim Flow 관리 Analysis

> 작성일: 2026-06-14
> 분석 기준: 현재 `index.html` 구현
> 분석 범위: `1. 접수 확인` 상세 팝업과 후속 단계 구현 준비 상태

## 1. 결론

`1. 접수 확인` 단계는 사용자가 요청한 목표 기준으로 구현 완료 상태다. 계약정보, 사고정보, 고객 정보 요청, 상담 메모까지 빠른 액션 팝업 흐름이 구성되어 있고, 단계 이동/이전 이동을 지원한다.

전체 Claim Flow 기준으로는 2~5단계 상세 팝업이 아직 미구현이므로 전체 기능은 부분 완료 상태다.

## 2. 매칭 상태

| 범위 | 상태 | 비고 |
|---|---:|---|
| 접수 확인 빠른 액션 순서 | 완료 | 계약정보 확인 → 사고정보 확인 → 고객 정보 요청 → 상담 메모 |
| 계약정보 확인 팝업 | 완료 | 자기부담금 단일 필드 및 Claim별 테스트데이터 반영 |
| 사고정보 확인 팝업 | 완료 | 처리 상태 제거, 완료 전 체크 상단, 사고조사 입력폼 구현 |
| 고객 정보 요청 팝업 | 완료 | 알림톡 현황 표, 휴대폰 메시지 UI, 템플릿 저장 |
| 상담 메모 팝업 | 완료 | 상담 분류, 메모, 이력, 완료 처리 |
| 입고 / 선견적 상세 팝업 | 미구현 | 후속 Claude Code 작업 범위 |
| 수리 승인 상세 팝업 | 미구현 | 후속 Claude Code 작업 범위 |
| 손해사정 상세 팝업 | 미구현 | 후속 Claude Code 작업 범위 |
| 지급 / 정산 상세 팝업 | 미구현 | 후속 Claude Code 작업 범위 |

## 3. 접수 확인 세부 분석

### 3.1 계약정보 확인

일치 항목:

- SK렌터카와 임차인 간 임대차 계약 정보 표시
- 렌트/리스, 상품구분, 계약번호, 관리지점, 최초등록일, 차량번호, 모델명, 잔존가액, 고객명 표시
- 사고대차/정비대차 포함 여부 표시
- `자기부담금` 단일 필드 표시
- Claim 테스트데이터별 자기부담금 값 표시

### 3.2 사고정보 확인

일치 항목:

- 사고조사 정보는 입력 가능한 필드로 구성
- 완료 전 체크를 상단으로 이동
- 처리 상태 박스 제거
- 사고내용 textarea 제공
- 사고유형, 경찰 신고 여부, 견인 여부는 select로 수정 가능

### 3.3 고객 정보 요청

일치 항목:

- 사고접수 관련 알림톡 발송 현황 표 표시
- 발송되지 않았거나 발송이 필요한 항목을 발송 가능
- 휴대폰 형태 메시지 작성 UI 제공
- 알림톡/문자 채널 선택
- 메시지 직접 입력 및 발송 가능
- 자주 쓰는 템플릿 저장 가능

### 3.4 상담 메모

일치 항목:

- 상담 분류 선택
- 상담 메모 직접 입력
- 자주 쓰는 상담 템플릿 적용
- 메모 저장 후 접수 확인 완료

## 4. 남은 리스크

| 리스크 | 영향 | 대응 |
---|---|---|
| 브라우저 자동 검증 미수행 | 실제 클릭 흐름에서 레이아웃 문제가 남을 수 있음 | 후속 작업 시작 전 접수 확인 회귀 테스트 수행 |
| 단일 파일 크기 증가 | 유지보수 난이도 상승 | 단계별 렌더링 함수를 명확히 분리 |
| 2~5단계 구현 시 접수 확인 기능 회귀 가능 | 완료된 1단계 품질 저하 | 접수 확인 회귀 테스트를 매번 수행 |
