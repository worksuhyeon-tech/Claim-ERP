# 이미지시스템 Analysis

> 작성일: 2026-06-14
> 기준: `이미지시스템.design.md` vs `index.html` 실제 구현
> 검증 방식: 정적(코드 대조) + JS 문법 검사 + Node 스텁 자료구조/로직 검증

## 1. 결론

Design 대비 구현 일치율 **약 96%**. Plan 성공기준(SC1~SC6) 모두 충족. 미세 차이는 "확대보기/다운로드"가 실제 동작이 아닌 데모 토스트라는 점(의도된 Out of Scope).

## 2. 매칭 상태

| 축 | 상태 | 비고 |
|---|---|---|
| 구조(Structural) | ✅ | `.app-shell/.side-nav/.app-main/#viewClaims/#viewImages` 태그 균형 확인 |
| 기능(Functional) | ✅ | 사이드바 전환·폴더탭·그리드·다중선택·문자발송·발송이력 구현 |
| 데이터/계약 | ✅ | `IMAGE_FOLDERS`(7), `CLAIM_IMAGES`(4건 폴더키 정합), `imageSendLog` 적재 |
| 런타임(수동) | ⚠️ | 브라우저 클릭 시나리오는 사용자 테스트 예정(자동화 환경 없음) |

## 3. 성공 기준 점검

| 기준 | 상태 | 근거 |
|---|---|---|
| SC1 화면 전환 | ✅ | `switchView()` + `.view[hidden]` + `#viewName` 갱신 |
| SC2 폴더 탭/장수 | ✅ | `renderImgMain()` 폴더 탭 + `folderImages().length` 배지 |
| SC3 SVG 썸네일/다중선택 | ✅ | `renderImageThumb()` + `imgSelected` Set 토글 |
| SC4 문자 발송/이력 | ✅ | `openImageSendModal()` 토글·미리보기, `sendImages()` 로그 적재(LINK/MMS 2건 검증) |
| SC5 기존 화면 회귀 없음 | ✅ | `#viewClaims` 마크업/이벤트 보존, 위임 이벤트 유지 |
| SC6 단일 파일/오프라인 | ✅ | 외부 네트워크 무의존, 인라인 SVG 플레이스홀더 |

## 4. 검증 로그

- `node --check`(스크립트 블록): 통과
- 자료구조: 폴더 7개, 4개 사고건 폴더키 누락 없음, 총 장수 6/7/9/8
- `findImage`/`folderImages`: 정상
- `sendImages` 로그: LINK·MMS 2건 정상 적재(`오세린 LINK 문자 2장`, `MMS 알림톡 2장`)
- HTML 구조 태그 개폐 균형 확인

## 5. 남은 리스크 / 갭

| 항목 | 심각도 | 처리 |
|---|---|---|
| 확대보기·다운로드가 데모 토스트 | Minor | Out of Scope(라이트박스는 후속) |
| 브라우저 실제 클릭 미검증 | Minor | 사용자 테스트로 확인 예정 |
| 민감 이미지 마스킹 없음 | Minor | 보안링크 발송으로 1차 완화, 마스킹은 후속 |
