# SK렌탈 Claim Flow 관리 화면 작업계획서

## 1. 프로젝트 개요

- 화면명: SK렌탈 Claim Flow 관리
- 목적: 접수부터 지급/정산까지 Claim 처리 흐름을 한눈에 보고, 담당자가 오늘 처리해야 할 업무를 빠르게 조치하는 SaaS형 ERP 데모 화면을 설계한다.
- 대상 사용자:
  - SK렌탈 Claim 담당자
  - Claim 운영 관리자
  - 신사업 영업직원
  - 고객사 임원

## 2. 화면 핵심 메시지

화면에서는 아래 메시지가 자연스럽게 느껴져야 한다.

1. 접수부터 정산까지 Claim 처리 현황이 한눈에 보인다.
2. 어느 단계에서 병목이 생기는지 바로 파악할 수 있다.
3. 담당자는 오늘 무엇을 처리해야 하는지 바로 알 수 있다.
4. 고객, 공업사, 손해사정, 정산 업무를 한 화면에서 연결할 수 있다.
5. 향후 Pro/Max 버전으로 ERP, 자동화, 정산관리까지 확장 가능해 보인다.

## 3. 전체 화면 구조

화면은 아래 4개 영역으로 구성한다.

1. 상단 업무 Flow 카드
2. 상단 보조 성과 지표
3. 좌측 조치유형 탭
4. 중앙 Claim 리스트 + 우측 빠른 처리 패널

레이아웃 구조:

- 상단: 업무 Flow 카드 5개를 가로 배치한다.
- 본문: 3단 구성으로 배치한다.
  - 좌측: 조치유형 탭
  - 중앙: Claim 리스트
  - 우측: 선택 건 상세 및 액션 패널

## 4. 상단 업무 Flow 카드 설계

상단 Flow는 아래 5단계로 구성한다.

1. 접수 확인
2. 입고 / 선견적
3. 수리 승인
4. 손해사정
5. 지급 / 정산

각 Flow 카드에는 아래 정보를 표시한다.

- 단계명
- 현재 미결 건수
- 지연/주의 건수
- 핵심 업무 2~3개
- 클릭 시 해당 단계 관련 리스트 필터링

카드 사이에는 화살표를 넣어 업무 흐름이 보이게 한다.

예시:

접수 확인 -> 입고 / 선견적 -> 수리 승인 -> 손해사정 -> 지급 / 정산

## 5. Flow 카드별 업무 내용

### 접수 확인

- 차량 및 계약정보 확인
- 이전 접수이력 확인
- 운전자 정보 및 사고정보 확인

### 입고 / 선견적

- 입고지 확인
- AOS 선견적 수신
- 선견적 검토

### 수리 승인

- 공업사 수리 오더
- 수리 승인
- 승인 후 진행상태 확인

### 손해사정

- AOS 청구 수신
- 손해사정
- 면부책 및 과실 최종 확인

### 지급 / 정산

- 손해액 확정 및 공업사 안내
- SK렌터카 전산 수리내역 입력
- 월 정산 데이터 정리

## 6. 보조 성과 지표

Flow 카드 아래에는 작은 글씨로 아래 문구를 표시한다.

오늘 발생 18건  면책 3건  종결 12건  평균 리드타임 2.8일

설명:

- 이 지표는 크게 강조하지 않는다.
- 담당자가 본인의 업무량과 종결 성과를 자연스럽게 확인하는 보조지표다.
- 평균 리드타임은 접수일부터 종결일까지 걸린 평균 소요일이다.

## 7. 좌측 조치유형 탭 설계

조치유형 탭은 아래 6개로 구성한다.

1. 고객 미응답
2. 고객 연락요청
3. 사진 미등록
4. 업체 미회신
5. 입고/수리 지연
6. 정산 대기

추가 요구사항:

- 좌측 탭 상단에 현재 선택된 Flow 단계를 표시한다.
- `전체 조치유형` 버튼을 제공한다.
- Flow 카드가 큰 업무 단계라면, 좌측 탭은 해당 단계 안의 조치유형 필터 역할을 한다.

## 8. 중앙 Claim 리스트 설계

중앙 리스트의 필수 컬럼은 아래와 같다.

1. 긴급도
2. 사고번호
3. 고객명
4. 차량번호
5. Flow 단계
6. 조치유형
7. 현재상태
8. 경과시간
9. 담당자
10. 처리상태

값 정의:

- 긴급도: 긴급 / 주의 / 정상
- 처리상태: 미처리 / 처리중 / 완료 / 보류
- Flow 단계: 접수 확인 / 입고선견적 / 수리 승인 / 손해사정 / 지급정산

## 9. 우측 빠른 처리 패널 설계

우측 패널에는 선택된 사고건의 상세 정보와 다음 액션을 표시한다.

공통 표시 항목:

- 사고번호
- 고객명
- 차량번호
- 고객구분
- 현재 Flow 단계
- 현재상태
- 조치유형
- 경과시간
- 담당자
- 조치 필요 내용
- 다음 권장 액션
- 빠른 액션 버튼
- 처리 메모 입력창
- 처리완료 버튼
- 보류 처리 버튼

## 10. Flow 단계별 액션 버튼

### 접수 확인

- 고객 정보요청
- 계약정보 확인
- 사고이력 조회
- 처리완료

### 입고 / 선견적

- 입고지 확인
- 선견적 확인
- 사진요청 발송
- 공업사 요청

### 수리 승인

- 수리 승인
- 공업사 오더
- 고객 안내발송
- 관리자 확인요청

### 손해사정

- AOS 청구 확인
- 손해사정 등록
- 면부책 확인
- 관리자 검토요청

### 지급 / 정산

- 손해액 확정
- 공업사 안내
- SK전산 입력
- 월정산 반영

## 11. 테스트 데이터 설계

`index.html` 구현 시 최소 12건의 더미 데이터를 사용한다.

각 데이터에는 아래 필드를 포함한다.

- 사고번호
- 고객명
- 차량번호
- 고객구분
- flowStage
- 조치유형
- 현재상태
- 긴급도
- 경과시간
- 담당자
- 처리상태
- 조치내용
- 다음 권장 액션

데이터는 아래 Flow 단계가 모두 포함되도록 구성한다.

- 접수 확인 2건 이상
- 입고 / 선견적 3건 이상
- 수리 승인 2건 이상
- 손해사정 2건 이상
- 지급 / 정산 3건 이상

## 12. 인터랙션 설계

`index.html` 구현 시 아래 인터랙션을 포함한다.

1. 상단 Flow 카드 클릭 시 해당 Flow 단계 데이터만 중앙 리스트에 표시
2. 좌측 조치유형 탭 클릭 시 현재 선택된 Flow 단계 안에서 해당 조치유형만 필터링
3. 전체보기 버튼 클릭 시 모든 Flow 단계 데이터 표시
4. 전체보기 상태에서 조치유형 탭 클릭 시 전체 데이터 중 해당 조치유형만 필터링
5. 중앙 리스트 행 클릭 시 우측 패널 데이터 변경
6. 선택된 Flow 카드, 선택된 탭, 선택된 행은 시각적으로 강조
7. 우측 액션 버튼 클릭 시 토스트 메시지 표시
8. 처리완료 버튼 클릭 시 해당 건의 처리상태를 완료로 변경
9. 보류 처리 버튼 클릭 시 해당 건의 처리상태를 보류로 변경

## 13. 디자인 방향

- 전체 배경: `#F6F8FB` 계열
- 메인 카드: 흰색
- 포인트 컬러: 신뢰감 있는 블루 계열
- 긴급 뱃지: 붉은색 계열
- 주의 뱃지: 주황색 계열
- 정상 뱃지: 초록색 또는 회색 계열
- 버튼은 너무 많아 보이지 않게 정돈
- 주요 버튼은 파란색
- 보조 버튼은 회색 테두리
- 전체적으로 B2B SaaS 데모 화면처럼 보여야 함
- 복잡한 보험사 업무시스템 느낌보다, 임원과 영업직원이 이해하기 쉬운 화면이어야 함

## 15. 테스트 계획

구현 완료 후 아래 시나리오로 테스트한다.

1. `index.html` 더블클릭
2. 상단 Flow 카드 중 `입고 / 선견적` 클릭
3. 중앙 리스트에 입고/선견적 단계 건만 표시되는지 확인
4. 좌측 탭에서 `업체 미회신` 클릭
5. `CLM-2026-0004` 선택
6. 우측 패널에서 AOS 선견적 수신 대기 내용 확인
7. `공업사 요청` 버튼 클릭 시 토스트 메시지 확인
8. `처리완료` 클릭 시 처리상태가 완료로 변경되는지 확인
9. `전체보기` 클릭 시 전체 데이터가 다시 표시되는지 확인

## 구현 전 확인사항

- 현재 루트에 기존 `index.html`이 있으므로, 다음 구현 단계에서는 신규 생성이 아니라 기존 파일을 교체 또는 수정할지 먼저 결정해야 한다.
- 테스트 시나리오의 `CLM-2026-0004` 데이터가 반드시 `입고 / 선견적` 단계와 `업체 미회신` 조치유형에 포함되도록 더미 데이터를 설계해야 한다.
- 단일 HTML 데모로 구현할 경우 외부 라이브러리 없이 HTML, CSS, JavaScript만 사용할지 확인해야 한다.
- 화면 목적이 영업용 데모인지, 실제 운영자 교육용 프로토타입인지에 따라 정보 밀도와 문구 톤을 조정해야 한다.
